#### 简单介绍



```
/：		linux的根目录，是linux所有文件的起点
/boot: 	 存储开机引导加载器和内核文件的目录
   vmlinuz-3.10.0-862.el7.x86_64 内核文件
    3.10.0-862 --> 主板号.次版本号.修正号
         
次版本号： 奇数表示测试版本
         偶数表示稳定版本
         修正号：修改次数  
         el7:red hat 7.x或centos7.x
         x86_64 64位操作系统
```



###### 目录结构：

```
/bin:    存放linux基本命令的目录,存放二进制文件
/sbin：  存放管理员使用命令的目录,存放二进制文件
/dev:    存放设备文件和特殊文件的目录
/etc:    存放配置文件的目录(存放纯文本文件)
/lib :   共享库文件的目录
/lib64：  64位系统的共享库文件的目录
/home：   普通用户的家目录
/root：  管理员root的家目录
/media:  移动设备的挂载点
/mnt :    临时设备的挂载点
/opt:     第三方应用程序的安装目录(弃用)
/proc:    用于输出内核与进程信息相关的虚拟文件系统
/run ：   运行中的进程相关的数据，通常用于存储过程的pid文件
/srv :    主要用来存储本机或本服务器提供的服务或数据
/sys ：   用于输出当前系统上硬件设备相关信息的虚拟文件系统
/tmp:     临时文件存放目录
/usr:     用户级文件存放目录
/usr/local : 第三方应用程序的安装目录

/var:     系统产生的不可自动销毁的缓存文件、日志记录
/var/log :  日志文件的存放目录
/var/mail:  邮箱文件的存放目录
```

```
linux的文件类型：
     (-)f:普通文件
     d: 目录文件
     b: 块设备文件
     c: 字符设备文件
     l: 符号链接文件
     p: 管道文件
     s: 套接字文件
```



指令主体 [选项] [操作对象]



```
ls 路径 ：列出该路径下的所有文件
ls 选项 路径 ：以指定选项展示路径下的文件
```

```
mkdir -p 路径 ：多层不存在的目录
mkdir 路径 路径 路径 ：创建多个目录
```

```
cp -r 目录 目录 ：复制文件夹
mv 目录 目录 ：移动目录
mv 文件 文件 ：移动文件或重命名
rm -rf 文件 ：f参数 不会一直询问(已测试，结果：不需要 f 也不会询问)
```

```
cat 文件 文件  > 文件 ： 文件合并
```
